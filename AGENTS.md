# AGENTS.md

## Project Overview

consultation-record is a POS (problem oriented system) application for analyzing and recording organizational issues in development teams.

The application records consultation cases, problems, observations, assessments, interventions, and follow-ups so the team can preserve the reasoning process behind organizational diagnosis.

## Current Technical Direction

- Java 25
- Spring Boot 4
- Thymeleaf server-rendered UI
- Spring Security form login
- PostgreSQL 18
- Testcontainers-managed PostgreSQL 18 for application and persistence tests
- Persistence mapper is not finalized; MyBatis and JPA are both under consideration
- Gradle wrapper is the canonical build entry point

## Build and Test

Use the Gradle wrapper:

```sh
./gradlew test
```

The test suite uses Testcontainers and expects Docker to be available.

On this macOS/Rancher Desktop environment, local test execution may require:

```sh
DOCKER_HOST=unix:///Users/ssobue/.rd/docker.sock \
TESTCONTAINERS_DOCKER_SOCKET_OVERRIDE=/var/run/docker.sock \
TESTCONTAINERS_RYUK_DISABLED=true \
./gradlew test
```

`TESTCONTAINERS_RYUK_DISABLED=true` is a local workaround. Prefer a Docker environment where Ryuk can run normally in CI.

## Java Version Notes

Builds may fail because of Java version mismatches. Do not conclude that the build is broken after trying only one JDK.

Use `/usr/libexec/java_home` to switch JDKs when needed. Example:

```sh
export JAVA_HOME="$(/usr/libexec/java_home -v 25)"
```

## Persistence Guidance

PostgreSQL 18 is fixed for the database.

The mapper strategy is intentionally still open:

- MyBatis is attractive for explicit SQL, custom timeline queries, and analysis/reporting screens.
- JPA is attractive for the first CRUD-heavy vertical slice and conventional parent-child aggregate relationships.

Do not make a broad MyBatis-to-JPA or JPA-to-MyBatis migration without first updating the design documents and the build dependencies together.

If JPA is selected:

- Use `spring-boot-starter-data-jpa`.
- Avoid exposing lazy-loaded entities directly to Thymeleaf templates.
- Keep schema management migration-driven, not Hibernate-DDL-driven.
- Make `@DataJpaTest` use Testcontainers PostgreSQL instead of an embedded database.

If MyBatis is selected:

- Keep SQL explicit and grouped by feature.
- Keep domain objects free from persistence framework annotations where practical.
- Test mapper SQL against Testcontainers PostgreSQL.

## Database Guidance

The current schema uses:

- `bigint GENERATED BY DEFAULT AS IDENTITY` primary keys
- `varchar` status fields with `CHECK` constraints
- `timestamp with time zone` audit columns
- cascade delete inside the case/problem record tree
- `RESTRICT` or `SET NULL` for user/team references

Do not replace this with PostgreSQL enum types, tenant tables, attachment tables, or triggers without recording the design decision.
